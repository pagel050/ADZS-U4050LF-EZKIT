Detailed Description

ADXL363 Accelerometer Driver (ADXL363)

This driver manages the ADXL363 accelerometer. The accelerometer is off-chip and requires some hardware knowledge to configure the driver correctly for use. GPIO connections are required if the ADXL363-generated interrupts are required. Additionally the connected SPI device number and chip select must be known.

Note
This driver requires the SPI driver. The application must include the SPI driver sources to avoid link errors.
This driver requires the DMA driver. The application must include the DMA driver sources to avoid link errors.
This driver requires the GPIO driver. The application must include the GPIO driver sources to avoid link errors.
The application must include drivers/adxl363/adi_adxl363.h to use this driver

enum ADI_ADXL363_INT

Enumeration of different sources for generation of the interrupts.

Upon getting an interrupt callback, the application should call the function adi_adxl363_GetIntSource to get the source(s) for the interrupt. Once the interrupt source is identified, application may have to call additional APIs to get more information related to the interrupt source. The applicable APIs are specified in the documentation of each interrupt source.If more there are more than one interrupt is registered, all interrupts are ORed and notified as single event.

Enumerator
ADI_ADXL363_INT_DATA_READY 

Data is ready to read.

This interrupt is generated when new data is available for reading. This will cleared when the data is read.

The data can be read using the blocking adi_adxl363_ReadData API.

ADI_ADXL363_INT_FIFO_READY 

FIFO contains some data that is available to read.

This interrupt is generated there is at least one valid sample available in the FIFO output buffer. This bit is cleared when no valid data is available in the FIFO. The data can be read using the blocking adi_adxl363_ReadFIFO API.

ADI_ADXL363_INT_WATERMARK 

FIFO watermark level reached.

This interrupt is generated when the number of entries in the FIFO has reached the specified watermark level. The watermark level can be set using API adi_adxl363_ConfigFIFO.

To clear the watermark interrupt, the application should read the data from the FIFO. The number of entries in the FIFO can be obtained using adi_adxl363_GetEntriesInFIFO.

ADI_ADXL363_INT_OVERRUN 

FIFO overrun occurred.

This interrupt is generated when hardware FIFO in ADXL363 has overflowed. To clear the overflow interrupt application should read the data from the FIFO. The number of entries in the FIFO can be obtained using adi_adxl363_GetEntriesInFIFO. The data can be read using the blocking adi_adxl363_ReadData API.

ADI_ADXL363_INT_ACTIVITY 

Activity detected.

This interrupt is generated when acceleration greater than the activity threshold is detected on any participating axis. The participating axis and threshold can be configured using adi_adxl363_ConfigActivity.

The first axis that contributed to the activity interrupt can be obtained by using the API adi_adxl363_GetActivityStatus.

ADI_ADXL363_INT_INACTIVITY 

Inactivity detected.

This interrupt is generated when the acceleration on all the participating axes is less than the inactivity threshold for more than the specified amount of time. The time, threshold and participating axes can be configured using adi_adxl363_ConfigInactivity.

ADI_ADXL363_INT_AWAKE 

Awake Interrupt.

When the ADXL363 detects activity, the awake bit becomes high.

enum ADI_ADXL363_FIFO_MODE

Enumeration of different modes the internal FIFO can operate.

Enumerator
ADI_ADXL363_DISABLE_FIFO 

In this mode, no data is stored in FIFO and any data already stored in it is cleared.

ADI_ADXL363_FIFO_OLDEST_SAVED 

In oldest saved mode, the FIFO accumulates data until it is full and then stops. Additional data is collected only when space is made available by reading samples out of the FIFO buffer. (This mode of operation is sometimes referred to as first N.)

ADI_ADXL363_FIFO_MODE_STREAM 

In stream mode, the FIFO always contains the most recent data. The oldest sample is discarded when space is needed to make room for a newer sample. (This mode of operation is sometimes referred to as last N.)

ADI_ADXL363_FIFO_MODE_TRIGGER 

In triggered mode, the FIFO saves samples surrounding an activity detection event.

uint32_t  nMemorySize,
ADI_ADXL363_HANDLE *  phDevice 
)

Opens the ADXL363 driver.

This opens the ADXL363 driver and upon success returns the handle to the driver. All the other APIs accept this handle as the parameter which determines the instance of the driver.

The ADXL363 uses SPI for communicating with the host processor.

After the ADXL363 driver is opened, the ADXL363 will be in standby mode. Once the required configuration is done, the API adi_adxl363_Enable should be called to enable the measurement.

Parameters
[in]pDevMemoryMemory required for ADXL363 device operation.
[in]nMemorySizeThe size of the memory passed in bytes.
[out]phDevicePointer to a location where the device handle is written.
Returns
Status
  • ADI_ADXL363_SUCCESS If successfully opened the device.
  • ADI_ADXL363_NULL_POINTER [D] If one of the given pointers are pointing to NULL.
  • ADI_ADXL363_INSUFFICIENT_MEMORY [D] If the given memory is not sufficient to operate the device.
  • ADI_ADXL363_DEVICE_ALREADY_OPENED [D] If the given device is already opened.
See also

ADI_ADXL363_RESULT adi_adxl363_ReadData ( ADI_ADXL363_HANDLE  hDevice,
void *  pBuffer 
)

Reads the sign-extended, 12-bit, accelerator output samples in blocking mode.

This is a blocking read call to get back the accelerator output samples. The samples read are the full 12-bit, sign-extended, samples.

This API should be called only once an event indicating that the data is ready to be read is notified. Otherwise there may be a chance of reading old data. Please follow the documentation of ADI_ADXL363_EVENT to know which events can be used to determine the availability of data.

The accelerator output is read in terms of triplets (data corresponding to x, y, z axis). As each sample is 16bit width, the size of the buffer should be multiple 6 bytes.

Parameters
[in]hDeviceHandle to ADXL363 device.
[in]pBufferPointer to the buffer where output samples are written. The buffer should be ADI_ADXL363_DATA_SIZE bytes in length.
Returns
Status
  • ADI_ADXL363_SUCCESS If successfully read the data.
  • ADI_ADXL363_INVALID_HANDLE [D] If the given ADXL363 device handle is invalid.
  • ADI_ADXL363_DEVICE_NOT_OPENED [D] If the given device is not yet opened.
  • ADI_ADXL363_DEVICE_ACCESS_FAILED [D] If failed to read/write device registers.
  • ADI_ADXL363_INVALID_OPERATION [D] If trying to read data when measurement is not enabled.
  • ADI_ADXL363_PENDING_IO [D] If trying to perform a read when another input/output operation is in progress.
  • ADI_ADXL363_SPI_DEV_FAILED If communication device related failure occurs.
