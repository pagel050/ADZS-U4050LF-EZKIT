Detailed Description

SPI-Flash driver for the Winbond W25Q32.

The driver provides high-level memory access functions that abstract the underlying communication protocol between the processor and the W25Q32.

Note
The application must include "drivers/spi-flash/w25q32/adi_w25q32.h" to use this driver. The application must include src/drivers/spi/adi_spi.c in order to avoid linker errors.

ADI_W25Q32_RESULT adi_w25q32_Open ( void *  pDevMemory,
uint32_t  nMemorySize,
ADI_W25Q32_HANDLE *  phDevice 
)

Initialize Driver.

Initializes the SPI driver for communication with the W25Q32

Parameters
[in]pDevMemory: Memory allocated for driver, must be ADI_W25Q32_MEMORY_SIZE bytes
[in]nMemorySize: Amount of memory allocated for driver
[out]phDevice: Pointer to the handle the application will use to reference the driver instance
Returns
ADI_W25Q32_RESULT
  • ADI_W25Q32_SUCCESS: Function call completed successfully.
  • ADI_W25Q32_FAILED_INSUFFICIENT_MEMORY: nMemorySize is too small.
  • ADI_W25Q32_FAILED_SPI_ERROR: SPI driver reported an error.
  • ADI_W25Q32_FAILED_BAD_PART_INFO: Did not read the expected part info, check SPI connection.
Note
This function must be called before any other functions are called

ADI_W25Q32_RESULT adi_w25q32_Write ( ADI_W25Q32_HANDLE  hDevice,
uint8_t *  buf,
uint32_t  bufSize,
uint32_t  address 
)

Write Data.

Prepares a transaction using the W25Q32 command + address protocol and sends it over the SPI bus, no data is received

Parameters
[in]hDevice: Device handle
[in]buf: Data to send (allocated by caller)
[in]bufSize: Number of valid bytes in buf (must be <= ADI_W25Q32_PAGE_SIZE)
[in]address: Address to write to (must be <= ADI_W25Q32_PAGE_SIZE * ADI_W25Q32_NUM_PAGES)
Returns
ADI_W25Q32_RESULT
  • ADI_W25Q32_SUCCESS: Function call completed successfully.
  • ADI_W25Q32_FAILED_SPI_ERROR: SPI driver reported an error.
  • ADI_W25Q32_FAILED_BAD_BUFFER_SIZE: Buffer size greater than one page.
  • ADI_W25Q32_FAILED_BAD_ADDRESS: Address greater than entire address space.
Note
User must be erase [address : address + bufSize] before writing
Can only write up to one page at a time for driver memory constraints
